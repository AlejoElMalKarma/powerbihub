Comprehensive Technical Glossary

Essential terminology from the Power BI ecosystem for every business user level.

Semantic Model

Formerly known as 'Dataset'. It is the underlying data structure, including tables, relationships, and measures.

Power Query

The data transformation engine (ETL) used to clean and shape data before it enters the model.

DAX

Data Analysis Expressions. A formula language used for custom calculations and dynamic metrics.

Drill-down

Navigating through a hierarchy in a single visual to see more granular data (e.g., Year > Quarter > Month).

Drill-through

Navigating from one report page to a separate, detailed page automatically filtered by a specific data point.

Cross-Highlighting

Interactivity where clicking a data point in one visual dims unrelated data in other visuals on the same page.

Import Mode

Data is copied into the Power BI in-memory engine. Fastest performance but requires a refresh cycle.

DirectQuery

Data remains in the source; Power BI queries it in real-time. Good for live data but can be slower.

Gateway

A secure bridge that allows the Power BI Service to access data located in on-premises servers.

Apps

A packaged collection of reports and dashboards designed for broad distribution to consumers.

Bookmarks

Saved states of a report page (filters, slicers, visual visibility) often used for storytelling or personal views.

RLS

Row-Level Security. A security layer that restricts data rows based on the user's role or identity.

Star Schema

A modeling best practice that separates numeric Fact tables from descriptive Dimension tables.

Smart Narratives

AI-driven text summaries that automatically highlight key insights and trends in a report.

OneLake

A centralized data repository (the OneDrive for data) where all organizational data artifacts are stored.
